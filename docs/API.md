# Organization Catalog API

## Overview

The FIDES Organization Catalog exposes a serverless API on Vercel that returns organizations with their ecosystem roles (issuers, credentials, wallets, relying parties).

## Endpoints

### `GET /api/public/organization`

Returns a paginated, filterable list of organizations.

**Query parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `search` | string | — | Full-text search on name, legal name, description |
| `country` | string | — | ISO 3166-1 alpha-2 country code filter |
| `role` | string | — | Ecosystem role filter: `issuer`, `credential`, `wallet`, `rp` |
| `certification` | string[] | — | Filter by one or more certifications (e.g. `iso27001`, `qtsp`). Repeat parameter for OR semantics |
| `sort` | string | `name` | Sort field: `name`, `country`, `updatedAt` |
| `direction` | string | `asc` | Sort direction: `asc`, `desc` |
| `page` | integer | `0` | Page number (0-based) |
| `size` | integer | `20` | Page size |

**Response shape:**

```json
{
  "content": [ /* Organization objects */ ],
  "totalElements": 104,
  "totalPages": 6,
  "number": 0,
  "size": 20
}
```

### `GET /api/public/organization/{id}`

Returns a **single** organization by catalog `id` (e.g. `org:animo`). The response body matches one element of the list `content` array. Encode reserved characters in the path (e.g. `org:animo` → `org%3Aanimo`).

### `GET /api/public/api-docs`

Returns the OpenAPI 3.1 specification as JSON.

## Deployment

The API is deployed on Vercel with the following configuration:

- **Build command:** `echo 'No build step; API only'`
- **Install command:** `npm ci`
- **Output directory:** `public`
- **Functions:** `api/**/*.ts` with 256 MB memory, 10s max duration

## Design choices

1. **Read-only** — the API only reads from `data/aggregated.json`, which is generated by the crawler
2. **In-memory cache** — aggregated data is cached for 60 seconds in the serverless function
3. **CORS enabled** — any origin can query the API
4. **CDN caching** — responses are cached at the edge for 5 minutes with stale-while-revalidate
